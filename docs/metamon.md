# Metamon

**Paper:** Human-Level Competitive Pokémon via Scalable Offline RL and Transformers (RLC 2025)
**Code:** https://github.com/UT-Austin-RPL/metamon
**Website:** https://metamon.tech
**Paper (arXiv):** https://arxiv.org/abs/2504.04395

## Overview

Metamon is an RL framework for Pokémon Showdown. It provides:
- 5M+ reconstructed human battle trajectories + 20M+ self-play trajectories
- Pretrained IL and RL policies (40+ baselines, 15M–200M params)
- Standardized team sets for training/evaluation
- RL training code built on [AMAGO](https://github.com/UT-Austin-RPL/amago) (off-policy Transformer RL)

Supported formats: Gen 1-4 OU, Gen 9 OU, Gen 9 VGC. Best models (Kakuna, Tauros) reach 90-99th percentile vs human players.

## Architecture

Metamon uses AMAGO, an off-policy Transformer RL agent:

```
Sequence of (obs, action, reward) timesteps → Transformer → Policy head + Value head
```

- **Observation:** Structured feature vector (not raw text) encoding battle state: active Pokémon stats, team roster, field conditions, move details
- **Action:** Discrete action IDs (move selection, switch, mega evolution, terastallize)
- **Reward:** Shaped reward combining damage, KOs, status, hazard, and win/loss
- **Sequence Model:** Transformer (with FlashAttention) or Mamba, processing full battle histories

### Training Pipeline (3 stages)

1. **Imitation Learning (IL) on human replays** — Supervised learning on reconstructed human trajectories. Models: `SmallIL` (15M), `MediumIL` (50M), `LargeIL` (200M)
2. **Offline RL on human replays** — Train actor-critic on the same human dataset. Models: `SmallRL`, `MediumRL`, `LargeRL`
3. **Self-play RL (Synthetic training)** — Use the IL+RL checkpoint to collect self-play battles, then train on combined human + synthetic data. Multiple rounds with increasing synthetic data. Final model: `SyntheticRLV2` (200M, trained on 1M human + 4M self-play)

### Dataset

- **Parsed replays:** 5.3M trajectories on HuggingFace
- **Raw replays:** 2.7M battles
- **Self-play:** 20M+ battles generated by Metamon policies
- **Teams:** 1000s of competitive teams from forums + procedural generation
- **Usage stats:** Showdown usage statistics per format

## How to Train Similar Models

### 1. Install

```bash
git clone --recursive git@github.com:UT-Austin-RPL/metamon.git
cd metamon
pip install -e .
# Also install amago
pip install -e amago
```

### 2. Set up Showdown server

```bash
cd server/pokemon-showdown
npm install
node pokemon-showdown start --no-security
```

### 3. Download data

```bash
export METAMON_CACHE_DIR=/path/to/storage
python -m metamon.data.download parsed-replays
```

### 4. Train a model

```bash
# IL training
python -m metamon.il.train --agent MediumIL --gens 1 --formats ou

# RL training
python -m metamon.rl.train --agent MediumRL --gens 1 --formats ou
```

### 5. Evaluate

```bash
python -m metamon.rl.evaluate --eval_type heuristic --agent Kakuna --gens 1 --total_battles 100
```

## Key Lessons for Hardread

### What Metamon does well
- **Scaled offline RL** from human demonstrations — IL first, then RL on the same data, then self-play
- **Dense reward shaping** — damage dealt/taken, KOs, status, hazards, setup moves
- **Structured observation** — feature vectors are more efficient for RL than text, but require custom encoders
- **Self-play curriculum** — start with human data (diverse, high-quality), then expand with self-play (targeted improvement)
- **AMAGO's sequence model** — processes full battles as sequences with Transformer memory

### What Hardread does differently (and advantages)
- **LLM-based policy** — Qwen3-4B with LoRA vs Metamon's custom Transformer — leverages pre-trained language understanding
- **GRPO for on-policy optimization** — Metamon uses off-policy actor-critic (AMAGO), Hardread uses GRPO which is simpler and directly optimizes the language model
- **Natural language observation** — markdown battle state vs structured features — more flexible but less efficient
- **OpenEnv standard** — portable to any OpenEnv-compatible infrastructure

### What Hardread could borrow from Metamon
1. **Self-play data collection** — use the SFT model to play thousands of battles against itself, then train GRPO on that data (instead of 20 battles)
2. **Better reward shaping** — add more reward components from Metamon (status, hazards, setup moves)
3. **Human replay dataset** — use Metamon's 5M+ parsed replays as a supervised finetuning dataset before GRPO
4. **Team diversity** — use Metamon's team sets for varied training opponents
5. **AMAGO's long-sequence handling** — if battles get long, Metamon's Transformer memory approach could help
